# 🌻 Интернет-магазин цветов: автотесты (архив)

Этот проект содержал автоматизированные UI-тесты для интернет-магазина цветов. Тесты разработаны на Python с использованием паттерна Page Object Model, Selenium WebDriver и фреймворка pytest. Однако **тесты больше не работают**, потому что структура и локаторы тестируемого сайта существенно изменились с момента последнего обновления проекта (30 июня 2025 года).

## ⚠️ Важное предупреждение

**Тесты нерабочие.** Код оставлен только для ознакомления с технической архитектурой и подходами к автоматизации. Для актуализации тестов требуется полный рефакторинг под новую версию сайта.

## 🛠 Используемые технологии

- **Python 3** – основной язык
- **Selenium WebDriver** – управление браузером
- **pytest** – фреймворк для написания и запуска тестов
- **Page Object Model** – структурирование кода
- **Allure** – генерация наглядных отчётов
- **Логирование** – для отслеживания выполнения тестов (папка `logs/`)
- **Скриншоты** – при падении тестов (папка `screens/`)

## 🚀 Как пробовали запускать (инструкция уже неактуальна)

Если вы хотите просто посмотреть код, выполните следующие шаги:

1. Клонируйте репозиторий:  
   ```bash
   git clone https://github.com/Waspish/senflowers_autotests.git
   cd senflowers_autotests
   ```
2. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv .venv
   .venv\Scripts\activate      # Windows
   source .venv/bin/activate   # macOS/Linux
   ```
3. Установите зависимости из `requirements.txt` (файл есть в репозитории):
   ```bash
   pip install -r requirements.txt
   ```
4. Запустите тесты командой:
   ```bash
   pytest
   ```
   или для генерации Allure-отчёта:
   ```bash
   pytest --alluredir=./test-results
   allure serve ./test-results
   ```

> **Обратите внимание:** из-за изменений на сайте тесты упадут с ошибками поиска элементов. Код приведён исключительно в образовательных целях.
